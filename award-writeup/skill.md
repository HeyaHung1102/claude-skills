# Skill: award-writeup

## 目的
高速公路局局務會報「頒獎/獻獎」背景資料生成器。
把臨時被要求準備的公文（背景說明 + 評定結果 + 局長致詞稿）從半天工作壓縮到審核 10 分鐘。

## 適用場景
- 局務會報前，綜合組要求各單位回報頒獎需求並檢附背景資料
- 任何需要「獎項背景說明 + 得獎名單 + 首長致詞稿」三段式公文的場合

## 輸入（你需要先準備好的資料）

```yaml
獎項名稱: ""           # 例：114年度金安獎選拔獎勵計畫
主辦單位: ""           # 例：路產組 / 綜合組 / 規劃組
起始年度: ""           # 例：107年開始頒發
辦理依據:
  - ""                # 法規條文或內部函文號
評選委員:
  人數: 0
  組成: ""             # 例：學者專家、機關代表
參獎數量:
  單位數: 0
  件數: 0
評分標準:               # 可留空，若有權重請列出
  - 項目: ""
    權重: ""
得獎名單:
  - 名次: "第一名"
    單位: ""
    得獎人: ""          # 姓名+職稱
    得獎內容: ""
    獲獎原因: ""        # 務必量化：筆數、百分比、達成率
    獎品內容: ""
```

## 輸出結構（固定骨架，不可省略段落）

```
一、獎項背景說明
(一) 頒獎/獎項緣由、沿革（自民國幾年開始辦理）
(二) 辦理依據/規定
(三) 評選委員組成、人數
(四) 參獎數量（人數、件數）
(五) 評選過程（含評分標準/權重，若有）

二、評定結果/得獎名單/獎品內容
[表格：名次｜單位｜得獎人員/領獎代表｜得獎作品名稱/內容｜獲獎原因｜獎品內容]

三、局長致詞稿（200-400字）
```

## 致詞稿寫作規則（從歷史範本歸納）

1. **開場**：肯定全體同仁/單位的付出（1-2句）
2. **中段**：具體點名前三名，每個獲獎理由都要附**量化成果**
   （例如：「辦理變更非公用財產達137筆，預算執行率達95%以上」）
   絕對不要用空泛形容詞取代數字
3. **結尾固定句式**：
   - 「期許大家持續以專業...」+ 對應業務領域的期許語句
   - 最後一句固定收尾：「謝謝大家。」
4. **字數**：嚴格 200-400 字，不附加超出範圍的內容

## 範例語料庫（已存於本資料夾 corpus/）
- 違章建築督導考核獎（114年度，路產組）
- 金安獎選拔獎勵計畫（114年度，綜合組）
- 交通工程環評追蹤考核優良獎（114年度，規劃組）

## 使用方式
把上面 YAML 範本填好實際資料，貼給 Claude，請求：
「依照 award-writeup skill 的格式骨架，生成完整背景資料 docx」
