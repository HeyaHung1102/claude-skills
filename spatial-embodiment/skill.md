# Skill: spatial-embodiment（空間整理 → 數位整理同構法）

## 目的
把「搬家後整理實體空間」的身體記憶，轉化成可重複套用的整理演算法，
同時適用於實體物品與數位檔案（repo / 硬碟 / 雲端）。
核心信念：**整理是一種演算法，不是一種天賦**。

## 六步驟口訣：「取分捨理位測」

| 步驟 | 實體動作 | 數位對應 | 驗收標準 |
|---|---|---|---|
| **取** 取出 | 把該區所有東西全部拿出來攤開 | 列出資料夾所有檔案（`ls -R` / tree） | 沒有隱藏、沒有堆疊死角 |
| **分** 分類 | 依「用途 + 頻率」分堆 | 依「主題 + 存取頻率」分群 | 每項都屬於且只屬於一堆 |
| **捨** 斷捨離 | 一年沒用 / 壞了 / 重複 → 丟 | 過期草稿、重複檔、失效連結 → 刪 | 留下的每一項都能說出「為什麼留」 |
| **理** 整理 | 縮小體積、直立擺放 | 扁平結構、前綴命名、去巢狀 | 每項第一眼可見（無需開三層） |
| **位** 歸位 | 高頻靠手邊、低頻上櫃 | 高頻放根目錄、冷資料進 archive/ | 動線頻率 = 歸位距離 |
| **測** 測試 | 閉眼能否 30 秒內找到 | 一週後能否憑記憶找到檔案 | 找不到 = 命名或位置錯，重來 |

## 三大原則（從實體整理歸納，可直接用於數位）

### 1. 先量容器，再決定內容
搬家時先量櫃深（例 67cm）、檯寬（例 1.59m），才決定放什麼。
→ 數位：先定 repo/資料夾結構（容器上限），再決定檔案進哪，
   不要先囤檔案再找地方塞。

### 2. 直立式存取 = 扁平可見性
平放疊三層 → 最底層永遠找不到。直立擺放 → 每項 O(1) 被看見。
→ 數位：避免五層巢狀資料夾；用「扁平 + 命名前綴」
   （例 `award_115_...`、`reply_115_...`）讓每個檔案一眼可辨。

### 3. 動線頻率決定歸位距離
每天用的（清潔劑、行動電源）放手邊；季用的上高櫃。
→ 數位：常用 skill / 進行中檔案放根目錄；
   冷資料、歷史範例進 `corpus/` 或 `archive/`。

## 使用方式（實體場景）
1. 選一個區塊（客廳 / 廁所 / 臥室，一次一區，勿跳）
2. 照「取分捨理位測」六步跑完
3. 每區跑完，記下「這區的高頻三件物品」→ 這決定下次歸位動線
4. 全部跑完後，回答一句話：「這次整理，我發現我最捨不得丟的是＿＿，為什麼？」
   （這句話往往指向真正的價值排序，對數位斷捨離同樣有效）

## 使用方式（數位場景，套用同一口訣）
- **取**：`find . -type f | sort`
- **分**：按主題建頂層資料夾（不超過 7 個，符合工作記憶上限）
- **捨**：刪重複 / 過期 / 失效；不確定的移到 `_pending_review/`
- **理**：檔名加日期或版本前綴，攤平巢狀
- **位**：README 放根、範例進 corpus、草稿進 drafts、結構化資料進 data
- **測**：一週後不看目錄樹，憑記憶說出某檔在哪；說不出就改名

## 與本 repo 現況的呼應
claude-skills 目前已隱含這套結構：
`skill.md`（根/高頻）＋`corpus/`（範例/中頻）＋`drafts/`（草稿）＋`data/`（結構化）。
這正是「動線頻率決定歸位距離」的實踐，可作為其他 skill 的資料夾範本。
