# 案例：2026/6 搬家回來後整理（第一個真實案例）

空間：約 4m×6m，隔成客廳 / 廁所 / 臥室。
狀態：搬家公司把東西載回，全堆在地上（六步中的「取」已完成，攤開待「分」）。

## 目視盤點（第二步「分」：依用途＋頻率）

| 堆別 | 物品 | 建議歸位空間 |
|---|---|---|
| 日用高頻 | 貓紋坐墊、黑色折疊椅、PHILIPS 家電、淨水器壓力桶（白色圓桶） | 客廳 / 廚房手邊 |
| 收納容器本身 | A4 紙箱、MUSELABS 箱、竹編提籃、黑色置物架+黑籃、五斗櫃、藤椅 | 先清空當容器，別當雜物堆疊平台 |
| 衣物寢具 | 花色棉被、西服衣物袋、行李箱 | 臥室 |
| 文具/雜項 | 膠帶、罐頭食品、牛皮紙捲、三腳架、禮盒紙袋 | 拆成「文具/食品/攝影」三小堆 |
| 待斷捨離候選 | 呼拉圈、多個紙袋提袋、未裝窗簾桿、野餐墊 | 進 `_pending`，一週後再決定 |
| 垃圾/回收 | 粉紅新北專用垃圾袋（已裝）、拆開空紙箱 | 直接出清 |

## 三原則在本案的落點
1. **五斗櫃/藤椅/黑置物架＝現成垂直容器**：先清空歸位，其他東西才有地方直立收納。
2. **粉紅垃圾袋+空紙箱先出清**：騰出地板動線，卡住走道的先移除，剩下的才看得清。
3. **窗簾桿/呼拉圈/野餐墊（季用或不確定）進 `_pending`**：一週沒用到就斷捨離。

## 下一步（六步剩下的「捨理位測」）
- 捨：垃圾袋+空箱出清、_pending 區設好
- 理：家電/寢具縮體積、容器直立擺放
- 位：高頻靠手邊、五斗櫃收低頻
- 測：一週後閉眼 30 秒找得到 = 過關

## 數位遷移發現
「先清空現成容器（五斗櫃）再收其他東西」＝ 數位上「先確立既有資料夾結構
再放新檔」，而不是每來一批東西就新開一個堆放點。搬家的地板動線 = repo 的根目錄，
卡住的垃圾袋 = 根目錄下該刪的過期檔，先清掉才看得清結構。
